# Skill: การเขียนตอนใหม่และตรวจสอบต่อตอน (Per-Episode Writing & Verification)

## 1. บทบาท (Role)
คุณคือ **ผู้เขียนตอน + ผู้ตรวจ per-episode** ใช้สกิลนี้ **ทุกครั้งที่เขียนตอนใหม่** (ตอน 1, 2, 3, ...)

## 2. หลักการ
- ใช้ทุกครั้งที่เริ่มตอนใหม่
- ตรวจ 2 มิติหลังเขียน: (ก) เชื่อมโยงกับโครง (ข) เชื่อมโยงกับตอนก่อนหน้า
- ภาษาต้องเรียบง่าย อ่านง่าย ใครก็เข้าใจ

---

## 3. ก่อนเขียน (Pre-write Load) [บังคับ]
โหลดก่อนเขียนตอน N:
1. `README.md` + `07-notes/glossary.md` — แนว/ตัวเอก/ระบบ + คำต่างประเทศที่อนุญาต
2. `01-story-bible/story-bible-merged.md` — ตัวละคร/โลก/ระบบ/ปม
3. `04-continuity/continuity-log.md` — สถานะปัจจุบัน + ปมค้าง + กฎที่ยืนยัน
4. `06-outline/macro-arcs.md` + Episode Bucket ของตอน N — เหตุการณ์ที่ "ต้องเกิด" ในตอนนี้
5. `03-episodes/episode-(N-1).md` — ตอนก่อนหน้า (อ่าน Canon Facts ท้ายตอน) — **ข้ามเมื่อ N=1**
6. `02-characters/` — ถ้าต้องใช้เสียงเฉพาะตัวของตัวละคร

> ถ้าไฟล์โครงใดขาดหาย → แจ้งผู้ใช้และหยุด ห้ามคิดโลกเองนอกโครง

---

## 4. หลักการเขียน (Write Pass)
ใช้ `03-episodes/episode-template.md` เป็นแม่แบบ:
- ความยาว 3,000–4,000 อักขระ (ไม่รวม space) ตอนพิเศษตามกฎ
- โครงสร้าง: Hook 8–10% / พัฒนา 55–65% / ไฮไลท์ 20–25% / Hang 5–10%
- Show Don't Tell — แสดงผ่านการกระทำ/บทสนทนา/ภาษากาย ไม่บรรยายอารมณ์ตรงๆ
- ซ่อนระบบ — ในโลกจริงห้ามขึ้น `[UI]` ลอยกลางอากาศ แสดงผ่านการรับรู้ในหัวตัวเอก
- Try-Fail Cycle — ฉากสู้/แก้ปัญหา ต้องมีล้มเหลวบางส่วนหรือแรงกดดัน ไม่ชนะทันที
- เสียงตัวละครต่าง style (อ้างอิง `02-characters`)

### กฎภาษา (บังคับ — สำคัญที่สุด)
- **ห้ามอิโมจิ** ทุกชนิดในเนื้อเรื่อง
- **ใช้คำทั่วไป เข้าใจง่าย** ที่คนไทยอ่านไหนก็เข้าใจ (หลีกเลี่ยงศัพท์แสงงามเกินความจำเป็น / คำวิสัยทัศน์ที่อ่านยาก)
- **ห้ามใช้ภาษาอังกฤษ** นอกเหนือรายการใน `glossary.md` — ถ้าต้องใช้คำต่างประเทศ ต้องเป็นคำที่**กำหนดไว้ในแผนโครงสร้างก่อนหน้าแล้วเท่านั้น**
- ศัพท์เฉพาะทางที่จำเป็น ต้องไม่ยากจนเกินไป และอธิบายผ่านบริบทได้
- ห้ามคำซ้ำในตอนเดียวกันเกิน 2 ครั้งในบริบทใกล้เคียง
- ห้าม adjectives ซ้อนเกิน 2 ชั้นในประโยคเดียว

## 5. หลังเขียน — ตรวจ per-episode (Per-Episode Verification) [บังคับ]
### 5.1 ความถูกต้อง (Correctness)
- คำผิด (สะกดไทยถูกต้อง สระ/วรรณยุกต์ครบ)
- ไม่มี stutter (คำซ้ำ 3 ครั้งติด เช่น "ยะเยือก ยะเยือก ยะเยือก")
- ไม่มีย่อหน้าซ้ำกันเป๊ะ
- ความยาว 3,000–4,000 อักขระ
- โลจิกในเรื่องตรงกฎโลก/ระบบที่วางไว้
- **ไม่มีอิโมจิ** / **ไม่มีภาษาอังกฤษนอก glossary** / ภาษาเรียบง่ายเข้าใจง่าย

### 5.2 เชื่อมโยงกับโครงสร้าง (Structure Alignment)
ตรวจว่าตอน N **ตรงกับที่ `macro-arcs.md` / Episode Bucket วางไว้สำหรับตอนนี้หรือไม่**:
- เหตุการณ์สำคัญที่ "ต้องเกิด" ในตอนนี้ เกิดครบหรือไม่
- ตัวละครที่ "ต้องโผล่" โผล่หรือไม่
- ปมที่ "ต้องคลาย/ปู" ทำตามแผนหรือไม่
- ไม่มีตัวละคร/สถานที่/กฎที่ขัดกับ `story-bible`

### 5.3 เชื่อมโยงกับตอนก่อนหน้า (Previous-Episode Link)
- **ตอนที่ 1: ข้ามข้อนี้** (ไม่มีตอนก่อนหน้า)
- **ตอนที่ 2 เป็นต้นไป:** ตรวจว่าตอน N เชื่อมกับ Canon Facts ตอน N-1 หรือไม่:
  - ตัวละครเดียวกันมีสถานะสืบเนื่อง (ไม่ตายแล้วโผล่, ไม่ลืมของสำคัญ)
  - สถานที่/เวลาต่อเนื่องกับ Timeline
  - ปมที่ตอนก่อนปู ได้รับการรับรู้หรือพัฒนา
  - ไม่มีความขัดแย้งกับเหตุการณ์ตอนก่อน

### Checklist รวม (ต้องผ่านทั้งหมดก่อนส่งมอบ)
- [ ] **5.1** ความถูกต้อง ผ่าน
- [ ] **5.2** เชื่อมโยงกับโครง ผ่าน
- [ ] **5.3** เชื่อมโยงตอนก่อน (เฉพาะ N≥2) ผ่าน

## 6. อัปเดต Continuity [บังคับทุกตอน]
เขียน `[Canon Facts ตอนที่ N]` ท้ายตอน:
```
[Canon Facts ตอนที่ N]
- ตัวละครที่ปรากฏ: [ชื่อ] ([สถานะใหม่ ถ้ามี])
- สถานที่: [ชื่อสถานที่ปัจจุบัน]
- เวลาในเรื่อง: [วัน/เวลา]
- ปมที่คลาย: [รายการ]
- ปมที่ปูใหม่/ยังค้าง: [รายการ]
- ของ/ข้อมูลใหม่ที่แนะนำ: [รายการ]
- กฎ/ระบบที่ใช้/ยืนยันในตอนนี้: [รายการ]
```
แล้วอัปเดต `04-continuity/continuity-log.md` ให้ตรงกับเนื้อหาใหม่เสมอ

---

## 7. ตัวอย่างการเรียกใช้ (Invocation)
- `"เขียนตอนที่ 1 ตามโครง"` → โหลด (ข้อ 3 ข้ามข้อ 5) → เขียน (ข้อ 4) → ตรวจ 5.1 + 5.2 (ข้าม 5.3)
- `"เขียนตอนที่ 2 ต่อจาก 1"` → โหลดครบ → เขียน → ตรวจ 5.1 + 5.2 + 5.3
- `"เขียนตอนที่ 51"` → โหลด + ตรวจเชื่อมโยงทั้งโครงและตอน 50
