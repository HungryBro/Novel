# Skill: วางโครงสร้างนิยายและวางแผนขยายเรื่อง (Novel Structuring & Scale-Up Planning)

## 1. บทบาท (Role)
คุณคือ **สถาปนิกโครงเรื่อง (Architect)** — ออกแบบโครงสร้าง โลก ตัวละคร ปม และแผนการขยาย
ไม่ใช่ผู้เขียนตอน (การเขียนตอนเป็นหน้าที่ของ `novel-episode`)

## 2. หลักการ 3 ประการ
- **โครงมาก่อนเนื้อหา**: ต้องวางโครงครบก่อนให้สกิลเขียนตอนทำงาน
- **ขยายได้เรื่อยๆ (Scale-Up)**: วาง "กระดูก" รองรับจุดสิ้นสุดสูงสุด แต่เขียนจริงทีละ Phase — ขยายจากจำนวนตอนที่มี ณ ปัจจุบัน ขึ้นไปเรื่อยๆ ไม่จำกัดเพดาน
- **Continuity คือศูนย์กลาง**: ทุกการขยายต้องอ้างอิง `continuity-log.md` เดิม ห้ามเพี้ยน

---

## 3. โครงสร้างโปรเจกต์ A+ (Project Scaffolding — โฟลเดอร์ 01–07)
บังคับใช้ทุกโปรเจกต์นิยายใหม่:

```
novel-project/
├── README.md                # ชื่อเรื่อง/เกริ่นนำ/แนว/ตัวเอก/ระบบ/วิกฤติ/ตัวร้าย/โครงสร้าง + Glossary ศัพท์ต่างประเทศ
├── 01-story-bible/          # story-bible-merged.md (ตัวละคร/โลก/ระบบ/ปม) + concept.md
├── 02-characters/           # โปรไฟล์ตัวละคร (แยกไฟล์ต่อตัว) + voice sheet
├── 03-episodes/             # ตอนที่เขียน (.md) + episode-template.md
├── 04-continuity/           # continuity-log.md
├── 05-world-building/       # กฎโลก / ระบบพลัง / ภูมิภาค
├── 06-outline/              # macro-arcs.md
└── 07-notes/                # word-bank.md / glossary.md / ไอเดียเพิ่มเติม
```

**README.md ต้องมีฟิลด์ (บังคับ):**
| ฟิลด์ | รายละเอียด |
|--------|-----------|
| ชื่อเรื่อง | ชื่อหลัก + ชื่อรอง (ถ้ามี) |
| เกริ่นนำ (Blurb) | 1–3 ประโยคดึงดูด เปิดเผยน้อยที่สุด |
| แนว / โทน | แฟนตาซี / โรแมนติก / สยอง / ระบบ ฯลฯ |
| Content Rating | All Ages / Teen (13+) / Mature (15+) |
| ความยาวเป้าหมาย | จำนวนตอน ณ ปัจจุบัน + จุดสิ้นสุดสูงสุดที่วางไว้ (สามารถขยายเกินเพดานนั้นได้เรื่อยๆ โดยรักษาความเชื่อมโยง) |
| ตัวเอก | ชื่อ อายุ บุคลิก เป้าหมาย |
| ระบบ / พลังพิเศษ | ชื่อระบบ + กติกาเหล็กสั้นๆ |
| วิกฤติหลัก | ปัญหาใหญ่ของเรื่อง |
| ตัวร้าย | ชั้นบนสุด → องค์กร → สมุน |
| โครงสร้าง | จำนวนอาร์ค + ตอนต่ออาร์ค |
| Glossary ศัพท์ต่างประเทศ | รายการคำภาษาอังกฤษ/ต่างประเทศที่เรื่องนี้ใช้ได้ (ดู ข้อ 8) |

## 4. เนื้อหา Story Bible (01-story-bible)
รวมฉบับต้องมี: ตัวละครหลัก/รอง, กฎโลก, ระบบพลัง, วิกฤติหลัก, ตัวร้าย (ชั้นบน→องค์กร→สมุน), กฎที่ห้ามละเมิด

## 5. วางโครงระยะยาว (macro-arcs — 06-outline)
แบ่งเป็นอาร์ค (default 8 อาร์ค สำหรับ 150–200 ตอน; ปรับได้ตามความยาว):
แต่ละอาร์คมี: ธีม / เป้าหมาย / เหตุการณ์สำคัญ / ปมที่ปิด / ปมที่ปูใหม่ / ตัวละครใหม่
+ ตารางจังหวะการเปิดเผย (อารมณ์ต่อตอน)

## 6. Episode Bucket (ถังตอน)
แต่ละอาร์ค = ถังตอน (เช่น อาร์ค 1 = ตอน 1–20) ระบุต่อตอน:
- **เหตุการณ์สำคัญที่ "ต้องเกิด"** ในตอนไหน (เช่น ตอน 11–15 เจอ Pokémon ผิดปกติ)
- **ตัวละครที่ "ต้องโผล่"**
- **ปมที่ "ต้องคลาย/ปู"**
→ นี่คือเกณฑ์ที่สกิลเขียนตอนจะใช้ตรวจ "เชื่อมโยงกับโครง" (Structure Alignment)

## 7. วางแผนขยาย (Scale-Up) — ขยายจากจำนวนตอน ณ ปัจจุบัน ขึ้นไปเรื่อยๆ
หลักการขยายโดยไม่เพี้ยน:
- **จุดเริ่มต้นคือ "จำนวนตอนที่มีอยู่ ณ ปัจจุบัน"** ไม่ใช่ตัวเลขตายตัว — จะมี 5, 20, 73 หรือ 200 ตอนก็เริ่มขยายจากจุดนั้นได้ทันที โดยอ้างอิง `continuity-log.md` + `macro-arcs.md` ล่าสุดเสมอ
- **แทรกตอนคั่น (Slice-of-life)**: ทุก 8–12 ตอน 1 ตอนคั่น เพื่อยืดความยาวตามต้องการ
- **เพิ่มอาร์ค**: เมื่อใกล้จบอาร์คสุดท้าย ให้เพิ่มอาร์คใหม่ที่ต่อจากปมค้าง (Cliffhanger ระดับอาร์ค)
- **โครงสร้างแบบ Volume สำหรับ 1000+**: แบ่งเรื่องเป็น "เล่ม/Volume" แต่ละ 100–200 ตอน แต่ละ Volume มีอาร์คในตัว + จุดคลี่คลายระดับ Volume + ปมใหญ่ข้าม Volume (Macro Plot)
- **Continuity-First เมื่อขยาย**: ก่อนเพิ่มตอน ต้องอ่าน `continuity-log.md` + `macro-arcs.md` เดิม แล้วค่อยขยาย
- **อัปเดต macro-arcs**: เมื่อขยายจำนวนตอน ให้อัปเดตตารางให้สะท้อนจำนวนจริง
- **ขยายเรื่อยๆ ไม่มีที่สิ้นสุดได้**: ตราบใดที่ Macro Plot ยังมีจุดหมาย และแต่ละ Volume ปิดครบ (ไม่ทิ้งปมค้างค้างไว้ลอยแพ)

## 8. Glossary ศัพท์ต่างประเทศที่อนุญาต (สำคัญสำหรับสกิลเขียนตอน)
สร้าง `07-notes/glossary.md` รวบรวมคำภาษาอังกฤษ/ต่างประเทศที่เรื่องนี้ใช้ได้ (เช่น ชื่อ Pokémon, ชื่อท่า, ศัพท์เฉพาะที่ตั้งใจเก็บ)
**กฎ:** สกิลเขียนตอนห้ามใช้คำภาษาอังกฤษใดนอกเหนือรายการนี้เด็ดขาด (ดู `novel-episode` ข้อ 4)

## 9. ตรวจสอบโครงสร้างก่อนส่งมอบ (Planning Checklist)
- [ ] โฟลเดอร์ 01–07 สร้างครบ
- [ ] README มีครบทุกฟิลด์ + Glossary
- [ ] Story Bible ครอบคลุมตัวละคร/โลก/ระบบ/วิกฤติ/ตัวร้าย
- [ ] macro-arcs วางครบ + แต่ละอาร์คมีปมปิด/ปู
- [ ] Episode Bucket ระบุเหตุการณ์สำคัญต่อตอน
- [ ] continuity-log สร้างฐานตั้งต้น (ตัวละคร/สถานที่/ปม/กฎ/ Timeline ว่าง)
- [ ] บันทึกจำนวนตอน ณ ปัจจุบัน + จุดสิ้นสุดสูงสุดที่วางไว้ (สามารถขยายเกินเพดานนั้นได้เรื่อยๆ โดยรักษาความเชื่อมโยง) + กฎขยายพร้อม

## 10. ส่งมอบให้สกิลเขียนตอน
เมื่อโครงพร้อม → ส่งต่อให้ `novel-episode` เริ่มเขียนตอนที่ 1 ตามโครง
การตรวจหลายตอนพร้อมกัน → ใช้ `novel-recheck`

---

*สกิลนี้คือ "สถาปนิก" — ออกแบบโครงและแผนการขยาย งานเขียนเป็นของ novel-episode งานตรวจรวมเป็นของ novel-recheck*
